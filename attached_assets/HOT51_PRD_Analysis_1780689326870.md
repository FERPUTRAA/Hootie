## 1. RINGKASAN EKSEKUTIF

HOT51 (alias **Scrollap1**) adalah platform streaming live agregator multi-layanan berbasis TypeScript fullstack. Proyek ini menggabungkan:

- **Backend Express.js** dengan 7 rute API khusus
- **Frontend React (Vite)** bergaya TikTok dengan infinite scroll
- **Cloudflare Worker** sebagai proxy CDN bypass
- **APK Android** yang telah di-decompile (64.2 MB, dari admin.down51apk.com)
- **Database PostgreSQL** dengan Drizzle ORM
- **AI Agent otonom** untuk pengembangan otomatis

### 2.2. Penyebab Utama Live Tidak Muncul (5 Bug Kritis)

#### 🔴 BUG #1 — IP Replit Diblokir ComHub
**File:** `artifacts/api-server/src/routes/live.ts` & `comhub.ts`

ComHub API (`www.comhub.live/vchat/app/live/livingList`) memblokir IP Replit secara langsung. Jika Cloudflare Worker belum di-deploy atau URL worker belum dikonfigurasi di env, semua request ke ComHub gagal dengan 403/timeout.

**Bukti dari README:**
> "Semua panggilan ComHub diarahkan melalui Cloudflare Worker (bypass geo-block)"

**Fix:**
```typescript
// comhub.ts — Pastikan CLOUDFLARE_WORKER_URL terset
const WORKER_URL = process.env.CLOUDFLARE_WORKER_URL;
if (!WORKER_URL) {
  throw new Error('CLOUDFLARE_WORKER_URL env tidak disetel! Deploy worker dulu.');
}
```

#### 🔴 BUG #2 — Kunci AES Decrypt Salah atau Tidak Cocok
**File:** `artifacts/api-server/src/routes/live.ts`

Stream HLS HOT51 dienkripsi dengan AES-128-CBC. Kunci dan IV hardcoded:
- **Key:** `star@livega*963.` (16 bytes)
- **IV:** `0608040307010502` (16 bytes)

Jika server HOT51 memperbarui kunci enkripsi, semua URL stream akan menghasilkan data korup dan video player gagal memuat.

**Gejala:** Video player menampilkan error "HLS manifest parse error" atau "decryption failed"

**Fix:**
```typescript
// Tambahkan fallback key dan logging
const STREAM_KEYS = [
  { key: 'star@livega*963.', iv: '0608040307010502' },  // Primary
  { key: process.env.STREAM_KEY_BACKUP || '', iv: process.env.STREAM_IV_BACKUP || '' }, // Fallback
];

async function decryptStreamUrl(encryptedUrl: string): Promise<string | null> {
  for (const { key, iv } of STREAM_KEYS) {
    try {
      const decipher = crypto.createDecipheriv('aes-128-cbc',
        Buffer.from(key, 'utf8'),
        Buffer.from(iv, 'utf8')
      );
      const decrypted = Buffer.concat([
        decipher.update(Buffer.from(encryptedUrl, 'base64')),
        decipher.final()
      ]);
      const url = decrypted.toString('utf8').replace(/\0/g, '');
      if (url.startsWith('http')) return url;
    } catch (e) {
      console.warn(`Decrypt gagal dengan key: ${key.substring(0,6)}...`);
    }
  }
  return null;
}
```

#### 🔴 BUG #3 — Proxy Pool Kosong / Semua Proxy Mati
**File:** `artifacts/api-server/src/routes/live.ts`

Proxy pool di-refresh dari ProxyScrape setiap 5 menit. Jika semua proxy dalam pool ditandai "dead" dan refresh gagal (karena rate limit atau timeout ProxyScrape), sistem tidak ada fallback — semua request live gagal.

**Gejala:** Log menunjukkan "All proxies dead, no fallback available"

**Fix:**
```typescript
// Tambahkan direct connection sebagai fallback terakhir
async function fetchWithProxyFallback(url: string, opts: RequestInit) {
  // 1. Coba proxy pool
  for (const proxy of getAliveProxies()) {
    try {
      return await fetchViaProxy(url, proxy, opts);
    } catch { markProxyDead(proxy); }
  }
  // 2. Coba Cloudflare Worker
  try {
    return await fetchViaWorker(url, opts);
  } catch (e) {
    console.warn('Worker failed:', e);
  }
  // 3. Direct request (last resort)
  console.warn('All proxies dead — trying direct connection');
  return await fetch(url, { ...opts, signal: AbortSignal.timeout(15000) });
}
```

#### 🔴 BUG #4 — Frontend Tidak Handle State "No Live Rooms"
**File:** `artifacts/tiktok-ui/src/`

Ketika API mengembalikan daftar live kosong (array `[]`), frontend kemungkinan tidak menampilkan pesan informatif — hanya loading spinner yang tidak pernah selesai.

**Gejala:** Layar putih atau spinner tak terbatas saat tidak ada room live tersedia

**Fix untuk React component:**
```tsx
// LiveFeed.tsx — Tambahkan state handling yang benar
const { data: rooms, isLoading, error } = useQuery({
  queryKey: ['live-rooms'],
  queryFn: fetchLiveRooms,
  refetchInterval: 30_000, // Refresh setiap 30 detik
});

if (isLoading) return <LoadingSpinner message="Memuat live..." />;
if (error) return <ErrorState message="Gagal memuat live. Coba lagi." onRetry={refetch} />;
if (!rooms?.length) return (
  <EmptyState
    icon="📡"
    title="Tidak ada live sekarang"
    message="Belum ada streamer yang live saat ini. Cek lagi nanti!"
  />
);
```

#### 🔴 BUG #5 — CORS Error dari Cloudflare Worker
**File:** `cloudflare-worker/hot51-proxy.js`

Worker mungkin tidak mengirim CORS headers yang lengkap untuk semua method, menyebabkan browser memblokir response dari worker.

**Gejala:** Console error "CORS policy: No 'Access-Control-Allow-Origin'"

**Fix untuk worker:**
```javascript
// hot51-proxy.js — CORS headers lengkap
const CORS_HEADERS = {
  'Access-Control-Allow-Origin': '*',
  'Access-Control-Allow-Methods': 'GET, POST, PUT, DELETE, HEAD, OPTIONS',
  'Access-Control-Allow-Headers': 'Content-Type, Authorization, token, userId, CheckSum, timestamp',
  'Access-Control-Max-Age': '86400',
};

export default {
  async fetch(request, env) {
    // Handle preflight
    if (request.method === 'OPTIONS') {
      return new Response(null, { headers: CORS_HEADERS });
    }
    // ... rest of proxy logic
    const response = await fetch(targetUrl, proxyRequest);
    const newHeaders = new Headers(response.headers);
    Object.entries(CORS_HEADERS).forEach(([k, v]) => newHeaders.set(k, v));
    return new Response(response.body, {
      status: response.status,
      headers: newHeaders,
    });
  }
};
```

## 5. DETAIL 7 RUTE API BACKEND

### 5.1. `live.ts` — Live Streaming HOT51 ⭐

**Endpoint:** `GET /api/live/rooms`

**Mekanisme:**
1. Request ke ComHub API via Cloudflare Worker
2. Parse daftar room aktif
3. Ekstrak URL stream dari field: `hlsStreamUrl`, `pullFlvUrl`, `pullHlsUrl`
4. Dekripsi AES-128-CBC jika URL terenkripsi
5. Return daftar room dengan URL stream yang sudah didekripsi

**AES Config:**
```
Algorithm : AES-128-CBC
Key       : star@livega*963.  (16 bytes UTF-8)
IV        : 0608040307010502  (16 bytes UTF-8)
```

**Proxy Pool:**
- Source: ProxyScrape API (refresh setiap 5 menit)
- Types: SOCKS5 → SOCKS4 → HTTP (urutan prioritas)
- Dead marking: proxy gagal 3x ditandai dead selama 10 menit
- Total timeout: 30 detik per request

### 5.2. `comhub.ts` — ComHub API Proxy

**Authentication Headers:**
```
token     : COMHUB_AUTH_TOKEN (env)
userId    : COMHUB_USER_ID (env)
timestamp : Unix timestamp (ms)
CheckSum  : MD5(token + timestamp)
```

**Endpoint ComHub yang digunakan:**
- `GET /vchat/app/live/livingList` — Daftar live aktif
- `GET /vchat/app/live/list?page=1&pageSize=20` — List dengan pagination

**CDN Domains yang diizinkan:**
- `cdnsi.com`, `livcdn.com`, `fsccdn.com`

### 5.3. `vava.ts` — WebSocket Vava

**Target:** `vbi.vervachat.com:443`

**Mekanisme:**
- Koneksi TCP raw → upgrade TLS → upgrade WebSocket manual
- Handshake dengan random `Sec-WebSocket-Key`
- Persistent background connection (survive browser tab close)
- Auto-reconnect: delay 3 detik (exponential backoff)
- Broadcast status via SSE ke semua klien

**Events SSE:**
- `ws_connected` — WebSocket berhasil terhubung
- `ws_message` — Pesan dari Vava server
- `ws_error` — Error koneksi

### 5.4. `autonomous.ts` — AI Agent

**Endpoint:**
- `POST /autonomous/read` — Baca file (max 8000 karakter)
- `POST /autonomous/write` — Tulis file ke direktori aman
- `POST /autonomous/apply-edit` — Edit presisi (search & replace)
- `POST /autonomous/run-command` — Jalankan perintah aman

**Direktori yang diizinkan untuk write:**
- `artifacts/tiktok-ui/src/`
- `artifacts/api-server/src/`
- `lib/`

**Perintah yang diizinkan:**
```
pnpm run typecheck
pnpm --filter <pkg> run <script>
pnpm install
pnpm run build
ls, cat, echo
```

**AI Model:** OpenRouter (Qwen) untuk konteks pengembangan

### 5.5. `apk.ts` — APK Decompile & Analisis

**Capabilities:**
- Decompile dengan Jadx (Java decompiler)
- Walk directory hasil decompile
- Baca file hasil decompile (max 12,000 bytes)
- Analisis kode via OpenRouter API
- Code review via CodeRabbit API (opsional)

### 5.6. `agora.ts` — Agora Token

**Endpoint:**
- `GET /agora/config` — App ID + status sertifikat
- `GET /agora/token?channel=X&uid=Y&expiry=Z` — Generate RTM token
- `GET /agora/channels?page=0&size=100` — Daftar channel aktif

### 5.7. `health.ts` — Health Check
