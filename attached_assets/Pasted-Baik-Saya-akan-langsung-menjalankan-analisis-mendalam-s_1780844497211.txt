## 1. KEY FINDINGS (5 Poin Utama)

Berdasarkan analisis mendalam terhadap `HOT51_FIX_REPORT.md`, `README.md` (PRD), dan struktur kode di kedua branch:

**① Geo-Block IP_LIMIT (KRITIKAL)** — Server Hot51 hanya mengizinkan IP dari Indonesia. Server Replit berbasis di AS dan selalu di-blok dengan error `G10001 / IP_LIMIT`.

**② Cloudflare Worker Tidak Meng-handle `OPTIONS` Preflight** — Worker lama melewatkan method `OPTIONS`, menyebabkan browser memblokir semua request cross-origin (CORS error di console).

**③ `hotFetch` di `live.ts` Tidak Agresif Proxying** — Pada branch main, backend mencoba request langsung ke Hot51 API terlebih dahulu, bukan melewati CF Worker. Ini selalu gagal karena IP Replit diblokir.

**④ Proxy Pool ProxyScrape Tidak Andal** — Proxy gratis dari ProxyScrape terlalu lambat/mati, menyebabkan kegagalan massal pada step enrichment `room-info`.

**⑤ CDN Allowlist di Worker Tidak Lengkap** — Domain CDN baru yang ditemukan dari analisis APK (`baccdn.com`, `hot51.live`) belum masuk allowlist worker lama, sehingga streaming URL tidak bisa di-proxy.

```javascript
// ✅ Handle OPTIONS untuk CORS preflight
// ✅ CDN allowlist lengkap termasuk domain baru dari APK
const ALLOWED_DOMAINS = [
  'hot51.app', 'cdnsi.com', 'livcdn.com', 'fsccdn.com',
  'baccdn.com',   // ← BARU: ditemukan dari analisis APK
  'hot51.live'    // ← BARU: domain CDN tambahan
];

const CORS_HEADERS = {
  'Access-Control-Allow-Origin': '*',
  'Access-Control-Allow-Methods': 'GET, POST, PUT, DELETE, OPTIONS',
  'Access-Control-Allow-Headers': 'Content-Type, sign, ac, token, Authorization',
  'Access-Control-Max-Age': '86400',
};

export default {
  async fetch(request, env) {
    // ✅ Tangkap OPTIONS preflight PERTAMA
    if (request.method === 'OPTIONS') {
      return new Response(null, { status: 204, headers: CORS_HEADERS });
    }

    const url = new URL(request.url);
    // ... forward ke Hot51 dengan semua header penting
    const targetResponse = await fetch(targetUrl, {
      method: request.method,
      headers: {
        ...Object.fromEntries(request.headers),
        'sign': request.headers.get('sign') || '',
        'ac': request.headers.get('ac') || '',
        'token': request.headers.get('token') || '',
      },
      body: request.method !== 'GET' ? request.body : undefined
    });
    
    return new Response(targetResponse.body, {
      status: targetResponse.status,
      headers: { ...Object.fromEntries(targetResponse.headers), ...CORS_HEADERS }
    });
  }
};
```

**Mengapa ini bekerja:** Browser melakukan `OPTIONS` preflight sebelum `POST`. Jika worker tidak merespons `OPTIONS` dengan `200/204`, browser langsung blokir request tanpa sempat menyentuh Hot51 API.

```typescript
// ✅ hotFetch gunakan CF Worker PERTAMA untuk POST request kritis
async function hotFetch(path: string, options: RequestInit = {}) {
  const isPostRequest = options.method === 'POST' || options.body;
  const workerUrl = process.env.HOT51_CF_WORKER_URL;

  // ✅ Step 1 (BARU): CF Worker untuk semua POST (room-info, dll.)
  if (workerUrl && isPostRequest) {
    try {
      console.log(`[hotFetch] Using CF Worker for POST ${path}`);
      const workerRes = await fetch(`${workerUrl}${path}`, {
        ...options,
        signal: AbortSignal.timeout(10000)
      });
      if (workerRes.ok) {
        console.log(`[hotFetch] CF Worker SUCCESS for ${path}`);
        return workerRes;
      }
      console.warn(`[hotFetch] CF Worker returned ${workerRes.status} for ${path}`);
    } catch (e) {
      console.error(`[hotFetch] CF Worker FAILED for ${path}:`, e);
    }
  }

  // Step 2: Proxy pool (untuk GET request atau fallback)
  for (const proxy of getActiveProxies()) {
    try {
      const proxyRes = await fetchViaProxy(proxy, path, options);
      if (proxyRes.ok) return proxyRes;
    } catch (e) {
      markProxyDead(proxy);
    }
  }

  // Step 3: Direct (terakhir, hanya untuk GET yang mungkin tidak diblokir)
  if (!isPostRequest) {
    return fetch(`https://api.hot51.app${path}`, {
      ...options,
      signal: AbortSignal.timeout(8000)
    });
  }

  throw new Error(`[hotFetch] All methods exhausted for ${path}`);
}
```

**Mengapa ini bekerja:** Endpoint `room-info` adalah POST request yang selalu diblokir jika tidak lewat Indonesian IP. Dengan CF Worker sebagai langkah pertama untuk semua POST, sistem bypass geo-block dengan konsisten.

Masalah utama di branch `main` adalah **kombinasi geo-blocking dan arsitektur proxy yang salah urutan prioritas**. Branch `fix-hot51-live-v2` memperbaiki kedua masalah ini: menempatkan CF Worker sebagai jalur utama untuk semua request POST kritis (`room-info`), dan menambahkan handler `OPTIONS` lengkap dengan CORS header yang benar ke Worker. Solusi lengkapnya: deploy Worker yang sudah diperbaiki, set `HOT51_CF_WORKER_URL` di Replit Secrets, lalu copy ketiga file fix ke branch `main`.
