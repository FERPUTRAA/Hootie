Server Replit berbasis di AS dan diblokir geo-restriction oleh Hot51 API (error G10001/IP_LIMIT). Ada 3 masalah yang harus diperbaiki sekaligus:

## PERBAIKAN 1 — `cloudflare-worker/hot51-proxy.js`

  'Access-Control-Allow-Methods': 'GET, POST, PUT, DELETE, OPTIONS, HEAD',
  'Access-Control-Allow-Headers': 'Content-Type, sign, ac, token, Authorization, userId, timestamp, CheckSum',

// Tambahkan domain baru ke allowlist CDN:

## PERBAIKAN 2 — `artifacts/api-server/src/routes/live.ts`

Ubah urutan logika di fungsi `hotFetch` (atau fungsi sejenis yang membuat HTTP request ke Hot51 API):

- SEBELUM: urutan = direct request → proxy pool → CF Worker
- SESUDAH: urutan = CF Worker (untuk POST) → proxy pool → direct request (hanya GET)

Logika barunya:
```typescript
async function hotFetch(path: string, options: RequestInit = {}) {
  const isPost = options.method === 'POST' || !!options.body;
  const workerUrl = process.env.HOT51_CF_WORKER_URL;

  // LANGKAH 1: CF Worker dulu untuk semua POST request
  if (workerUrl && isPost) {
    try {
      const res = await fetch(`${workerUrl}${path}`, {
        ...options,
        signal: AbortSignal.timeout(10000)
      });
      if (res.ok) return res;
      console.warn(`[hotFetch] CF Worker status ${res.status} untuk ${path}`);
    } catch (e) {
      console.error(`[hotFetch] CF Worker error untuk ${path}:`, e);
    }
  }

  // LANGKAH 2: Proxy pool sebagai fallback
  for (const proxy of getActiveProxies()) {
    try {
      const res = await fetchViaProxy(proxy, path, options);
      if (res?.ok) return res;
    } catch { markProxyDead(proxy); }
  }

  // LANGKAH 3: Direct hanya untuk GET
  if (!isPost) {
    return fetch(`https://api.hot51.app${path}`, {
      ...options,
      signal: AbortSignal.timeout(8000)
    });
  }

  throw new Error(`[hotFetch] Semua metode gagal untuk ${path}`);
}
```

Sesuaikan nama fungsi dan cara akses proxy pool dengan kode yang sudah ada di file tersebut. Jangan ubah logika AES decrypt, struktur route, atau endpoint lainnya.
